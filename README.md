# Shadow Cannon 🏹💥

Hi! I’m Susanna, and this is **Shadow Cannon**, a physics-based strategy game I developed with my teammate Claudia as part of our Bachelor in Artificial Intelligence.  

Shadow Cannon lets you step into the role of the last Master of the Shadow Cannon. You journey through four different worlds (Hell, Abyss, Cursed Lands, and Space) using bombs, shotguns, and lasers to defeat darkness threatening entire realms. Each level introduces new mechanics and challenges, making the game progressively more complex and exciting.  

---

## 🛠️ Technologies & Tools
- **Python 3.12.0** – main programming language  
- **Kivy 2.3.1** – for the graphical interface  
- **PyCharm** – IDE we used for development   

---

## 🎮 Features
- Four levels, each with unique visual themes and increasing difficulty  
- Multiple projectile types (bomb, shotgun, laser) with different mechanics  
- Physics-based gameplay: parabolic motion, collisions, laser reflections  
- Hall of Fame to track high scores and encourage replay  

---

## 🚀 Development Process
We developed Shadow Cannon step by step:  
1. **Prototype:** we built a basic cannon, projectile, and target to test ideas.  
2. **Physics & Collisions:** implemented parabolic motion, collision detection, and basic obstacles.  
3. **Level Expansion:** added new levels, projectiles, enemies, and mirrors incrementally.  
4. **Optimization:** balanced difficulty, fixed bugs, and polished graphics & sounds.  

Separating **logic** (Python code) from the **interface** (KV files) helped us keep the code clean and easy to modify.  

---

## ✨ What I Learned
During this project, I really learned how important it is to keep code modular and easy to maintain. Working step by step, testing each part before moving on, helped me understand the benefits of incremental development. I also discovered a lot about level design (how to balance difficulty so the game stays challenging but fair). Applying physics concepts like projectile motion, collisions, and laser reflections in practice was both fun and educational. On top of that, collaborating with my teammate showed me how much teamwork and exchanging ideas can improve not just the code, but also the creativity behind the game
